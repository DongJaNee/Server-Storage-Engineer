## **xFusion 주제**

##### 1. 서버
- 서버 기술은 다양한 소프트웨어와 하드웨어 기술을 포함하며, 이를 통해 서버가 효과적으로 동작하고 다양한 기능을 수행할 수 있게 한다. 
1)  하드웨어(HardWare) : 서버의 물리적 구성 요소로, CPU, 메모리, 저장 장치, 네트워크 카드 등을 포함합니다.
2)  운영 체제(Operating System) : 서버를 운영하고 관리하는 소프트웨어 환경으로, Linux, Windows Server 등이 있다.
3)  가상화(Virtualization) : 여러 운영 체제나 애플리케이션을 하나의 물리적 서버에서 동시에 실행할 수 있게 하는 기술로,  VMware, Hyper-V 등이 있다.
4)  네트워킹(Networking) : 서버 간의 데이터 통신을 관리하고 제어하는 기술로, 이더넷, 무선 네트워크, SDN(Software Defined Networking) 등을 포함한다.
5)  보안(Security) : 서버와 데이터를 보호하기 위한 기술로, 방화벽, 암호화, 침입 탐지 시스템 등이 있다.

##### Server Architecture(서버 아키텍처) 
- 서버 아키텍처는 서버 시스템의 설게와 구조를 의미하며, 이를 통해 효율적이고 확장 가능한 서버 환경을 구축할 수 있다.
1) 단일 서버 아키텍처(Single Server Architecture) : 모든 서비스를 하나의 서버에서 운영하는 구조로, 작은 규모의 서비스에 적합하다.
2) 다중 서버 아키텍처(Multi-Server Architecture) : 여러 서버에 서비스를 분산하여 운영하는 구조로, 확장성과 가용성을 높일 수 있다.

##### 2. CPU
- 중앙 처리 장치로, 서버에서 명령을 실행하고 데이털르 처리하는 핵심 부품이다.
1) 연산 수행 : 복잡한 수학적, 논리적 연산을 바르게 처리한다.
2) 명령어 실행 : 서버 소프트웨어 및 애플리케이션이 제공하는 명령어를 실행한다.
3) 멀티태스킹 : 다중 작업을 동시에 수행할 수 있도록 지원한다.
4) 데이터 이동 : 메모리와 저장 장치 간의 데이터를 신속하게 이동시킨다.
※고성능 서버 CPU는 일반적으로 클럭 속도, 코어 수, 캐시 메모리 크기 등의 특징을 통해 성능을 평가한다. 이를 통해 서버는 높은 처리 속도와 효율성을 유지할 수 있다. 
##### 3. 하드 디스크
- 컴퓨터와 서버에서 데이터를 저장하는 주요 장치 중 하나이다.
- 하드디스크는 회전하는 디스크(플래터)와 그 위에 데이터를 읽고 쓰는 헤드로 구성되어 있다.
1) 데이터 저장 : 대용량 데이터를 안정적으로 저장할 수 있다.
2) 데이터 엑세스 : 파일과 프로그램을 빠르게 읽고 쓸 수 있다.
3) 백업 및 복구 : 중요한 데이터를 백업하고 필요시 복구할 수 있다.
##### 4. RAID
- 여러 하드디스크를 하나의 논리적 단위로 결합하여 데이터 저장 성능, 가용성, 신뢰성을 향상시키는 기술이다.
1) RAID 0 (스트라이핑) : 데이터를 여러 디스크에 분산하여 읽기/쓰기 성능을 높인다.
   중복성이 없어 하나의 디스크가 고장 나면 데이터가 손실된다. (가속)
2) RAID 1 (미러링) : 데이터를 두 개의 디스크에 동일하게 저장하여 중복성을 제공한다.
   하나의 디스크가 고장 나도 데이터가 손실되지 않는다. (안전성)
3) RAID 5 (스트라이핑 + 패리티) : 데이터를 스트라이핑하고 패리티(오류 검출 코드)를 추가하여 성능과 중복성을 모두 제공한다.
   3개의 디스크가 필요하다.
4) RAID 6 (이중 패리티) : RAID 5와 유사하지만 패리티 블록을 두 개 저장하여 디스크 두개가 고장나도 데이터를 복구할 수 있다.
5) RAID 10 (RAID 1 + RAID 0) : 스트라이핑과 미러링을 결합한 방식으로, 성능과 중복성을 모두 제공한다.
   최소4개의 디스크가 필요하다.

### **서버 유형**
##### 1. 랙 서버(Rack Server) 
- 데이터 센터나 서버 룸에서 표준화된 랙(rack)이라고 불리는 프레임에 장착되는 컴퓨터 서버이다.
- 일반적으로 1U, 2U, 4U와 같은 표준 높은 단위로 제공되며, 각 "U"는 1.75인치(44.45mm)를 나타낸다.
1) 공간 효율성 : 많은 서버를 하나의 랙에 수직으로 배치하여 공간을 절약한다.
2) 확장성 : 필요한 만큼의 서버를 추가하거나 제거할 수 있어 유연한 확장이 가능하다.
3) 유지보수 용이성 : 랙 서버는 일반적으로 전면과 후면에서 액세스할 수 있어 유지보수가 용이하다.
4) 냉각 효율성 : 통합된 냉각 시스템을 통해 효율적으로 열을 관리한다.
   
##### 2. 이종 서버(Heterogeneous Server)
- 서로 다른 하드웨어 아키텍처나 운영 체제를 사용하는 여러 서버를 함께 사용하는 시스템을 의미한다.
1) 다양한 플랫폼 지원 : 이종 서버는 다양한 하드웨어 플랫폼과 운영 체제를 지우너하여 유연성을 제공한다.
2) 최적의 성능 : 각 서버는 특정 작업에 최적화되어 있어, 전체 시스템의 성능을 향상시킬수 있다.
3) 고가용성 : 여러 서버가 서로 다른 하드웨어와 소프트웨어를 사용하므로, 단일 서버 고장으로 인한 영향을 최소화할 수 있다.
##### 3. 고밀도 서버9Hight Density Server) 
- 작은 공간에 다수의 서버를 집적하여 효율적으로 설치하고 운영할 수 있도록 설계된 서버 유형이다.
- 주로 데이터 센터나 클라우드 서비스 제공 업체에서 사용되며, 높은 성능과 효율성을 제공하는 것이 특징.
1) 공간 절약 : 많은 서버를 작은 공간에 배치하여 물리적 공간을 절약한다.
2) 에너지효율성 : 에너지 소비를 최소화하면서 높은 처리 성능을 제공한다.
3) 유지보수 용이성 : 모듈식 디자인으로 유지보수가 쉽고 빠르게 이루어질 수 있다.
4) 확장성 : 필요한 만큼의 서버를 추가하거나 제거할 수 있어 유연한 확장이 가능하다.

### **관리 및 소프트웨어**
##### 1. IBMC(Integrated Baseboard Management Controller)
- 지능형 보드 관리 컨트롤러로, 서버나 네트워크 장치를 원격으로 관리할 수 있도록 하는 임베디드 하드웨어 기반 시스템.
1) 원격 전원 관리 : 전원 켜기/끄기, 강제 종료 및 시스템 재부팅 등을 원격으로 수행할 수 있다.
2) 시스템 모니터링 : CPU, 메모리, 디스크 등의 성능을 모니터링하고 알람을 발생시킬 수 있다.
3) 통합 관리 솔루션 : WISE-DeviceOn과 같은 소프트웨어 플랫폼과 통합되어 네트워크 내 모든 장치를 관리할 수 있다.
※어드밴텍, 휴와이, 인텔 등 여러 기업에서 IBMC를 제공하며, 다양한 산업 애플리케이션에서 활용된다. 
##### 2. FusionDirector
- Huawei에서 개발한 지능형 서버 관리 소프트웨어.
- 서버의 전 생애 주기를 관리하는 데 도움을 주며, 다양한 지능적 운영 및 유지관리기능을 제공한다.
  1) 지능적 유지보수 : AI기반의 메모리 고장 자동 복구 및 디스크 고장 예측 기능을 제공하여 시스템의 가용성을 높인다.
  2) 지능적 업그레이드 : 일괄 업그레이드를 지원하여 운영 비용을 절감하고 업그레이드 효율성을 높인다.
  3) 지능적 발견 : 서버 자산을 자동으로 인벤트하고 실시간으로 추적하여 공간 이용률을 향상시킨다.
  4) 지능적 에너지 절약 : 에너지 효율성을 높여 전체 비용을 줄인다.
  5) 지능적 배포 : 다양한 운영 체제를 지원하며, 하드웨어 계획에서 소프트웨어 배포까지 효율적으로 수행한다.

### **배포 및 구성**
##### 1. 단일 노드 배포 - 초기 구성(Single-Node Deployment-Initial Configuration)
- 단일 서버 또는 노드에서 OpenShift클러스터를 설치하고 구성하는 것을 의미.
- 주로 소규모 개발 환경이나 테스트 환경에서 사용
1) 필수 구성 요소 확인 : 단일 노드 OpenShift클러스터를 설치하기 위한 최소 시스템 요구 사항을 확인한다.
  ex) CPU, 메모리, 디스크 공간등
2) 네트워크 설정 : 네트워크 환경을 구성하여 클러스터가 우너활하게 통신할 수 있도록 한다.
  ex) DNS설정, IP주소
3) 설치 미디어 준비 : OpenShift설치를 위한 ISO 이미지 또는 기타 설치 미디어를 준비한다.
4) 클러스터 설치 : 설치 스크립트 또는 설치 관리 도구를 사용하여 OepnShift 클러스터를 설치한다.
  이 과정에서 클러스터 설정 파일을 통해 다양한 설정을 지정할 수 있다.
5) 클러스터 구성 : 설치가 완료된 후 클러스터의 다양한 구성 요소를 성정하고 조정한다.
  ex) 인증, 권한 부여, 네트워크 설정
6) 확인 및 테스트 : 클러스터가 올바르게 작동하는지 확인, 테스트 수행 
##### 2. 일괄 배포 - 초기 구성(Batch Deployment - Initial Configuration)
- 소프트웨어나 애플리케이션을 한 번에 여러 시스템 또는 환경 배포하는 것을 의미한다.
1) 사전 준비
2) 환경 설정
3) 배포 스크립트 작성
4) 테스트 및 검증
5) 배포 수행
6) 모니터링 및 유지보수
##### 3. 운영체제 설치(OS Installation)
1) 설치 미디어 준비
2) 부팅 순서 설정
3) 설치 미디어로 부팅
4) 운영체제 설치 과정 진행
5) 드라이버 및 업데이트 설치
6) 기본 소프트웨어 설치
